스프링 프레임워크 : 자바로 구성된 틀 / 뼈대
(Spring Framework)
		=> 소프트웨어적 관점	:	기능이나 클래스, 인터페이스 등을
											어느정도 만들어서 제공하는 반제품 (밀키트)
		=> "스프링" 자바-웹 애플리케이션을 개발하기위한 오픈소스 프레임워크
						(경량 프레임워크/톰켓 - 서블릿 컨테이너)
		
* 스프링의 특징
 - 	EJB(중량프레임워크) -> spring(경량프레임워크)
 - 	제어의 역행(IoC, Inversion of Control)기술을 사용해서 애플리케이션간의
	느슨한 결합을 제어가능
 - 의존성 주입(DI, Dependency Injection)기능을 지원(사용가능)
 - 관점지향 프로그래밍(AOP, Aspect-Oriented Programming)을 사용한 자원관리
 
 * 스프링의 주요기능
 -	Core : 다른 기능과 설정을 분리하기위해서 IoC기능을 제공
 -	Context : 스프링의 기본기능,
 				애플리케이션의 각 기능을 하는 객체(빈Bean)에 접근하는 방법제공
 -	DAO : JDBC기능을 조금더 편리하게 제공
 -	ORM : 하이버네이트, 마이바티스와 같은 (영속성)프레임워크를 연동가능
 -	AOP : 관점 지향프로그래밍 지원
 - WEB : 웹 개발에 필요한 기능 제공
 - WebMVC : 스프링 MVC 구현을 위한 기능을 제공
